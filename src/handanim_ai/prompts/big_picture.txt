## What animation tools are available

- `SketchAnimation`

- then fills if the drawable has a fill
- can delay fill using `wait_before_fill`
- can show a glowing pen tip when the drawable uses `glow_dot_hint`

### Appear / disappear
- `FadeInAnimation`
- `FadeOutAnimation`
- `ZoomInAnimation`
- `ZoomOutAnimation`

### Motion
- `TranslateToAnimation`
- `TranslateFromAnimation`
- `TranslateToPersistAnimation`

### Composition
- `CompositeAnimationEvent`
- `DrawableGroup`

Use these when:
- several objects should animate together

## Styling vocabulary

### Stroke style
Use `StrokeStyle` for:
- line color
- stroke width
- opacity

Safe default:
- black stroke, width around `2`

### Fill style
Use `FillStyle` for:
- shape color
- opacity
- fill pattern
- hachure spacing

Common fill patterns:
- `solid`
- `hachure`
- `hatch`
- `zigzag`

Great default for educational sketches:
- light fill color
- opacity around `0.2` to `0.5`
- `hachure` with moderate gap

### Sketch style
Use `SketchStyle` for:
- roughness
- bowing
- jitter / hand-drawn feel

Good default range:
- roughness `1` to `3`
- bowing `1` to `2`

## Planning guidance for strong educational scenes

### Recommended structure
1. Intro / title scene
2. Concept-building scenes
3. Relationship / transformation scenes
4. Worked example or algorithm scene
5. Final summary scene

### Timing heuristics
- simple sketch: `1` to `2` seconds
- complex sketch: `2` to `5` seconds
- fades: `0.4` to `1.0` seconds
- translations: `1` to `2` seconds
- leave short pauses between major conceptual beats
- keep the total runtime realistic for the topic

### Composition heuristics
- group related elements
- avoid overcrowding the frame
- prefer left-to-right or top-to-bottom explanatory flow
- use arrows and connectors to show causality
- use erasing or fade-out when changing topics
- keep labels close to the objects they explain
- reserve strong colors for emphasis, not everything

### Topic-specific planning suggestions
- For math: use `Math`, supporting `Text`, arrows, boxes, and step-by-step transformations.
- For science: use labeled diagrams, color-coded parts, and motion to show process.
- For CS / algorithms: use flowcharts, boxes, arrows, input/output nodes, and staged highlighting.
- For conceptual topics: use icons, grouped labels, comparisons, and progressive reveal.
