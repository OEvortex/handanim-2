## Handanim quick model

- `Scene` holds drawables and animation events.
- `ThreeDScene` extends `Scene` with a camera, 3D projection, fixed overlays, and camera-motion helpers.
- `scene.add(event, drawable)` schedules a drawable on the timeline.
- Time is **absolute seconds from video start**.
- Drawables stay visible after creation unless faded, zoomed out, or erased.
- Standard canvas: `Scene(width=1920, height=1088, fps=24)`.
- Default viewport maps roughly `1777 x 1000` world coordinates to `1920 x 1088` pixels.

## Core API

### Scene

- `Scene(width=1280, height=720, fps=24, background_color=(1, 1, 1), viewport=None)`
- `ThreeDScene(width=1280, height=720, fps=24, background_color=(1, 1, 1), viewport=None, camera=None)`
- `SpecialThreeDScene(...)` is a convenience subclass with helpers like `get_axes()` and `get_sphere()`.
- Main methods:
  - `scene.add(event, drawable)`
  - `scene.render("output.mp4", max_length=None)`
  - `scene.render("output.gif", max_length=None)`
  - `scene.render_snapshot("frame.svg", frame_in_seconds=5, max_length=None)`
- Use `scene.drawable_cache` when creating an `Eraser`.
- 3D-specific methods:
  - `scene.add_mobjects(*drawables, start_time=0.0, end_time=None)`
  - `scene.add_fixed_in_frame_mobjects(*drawables)`
  - `scene.add_fixed_orientation_mobjects(*drawables, point=None, points=None, center_func=None, use_static_center_func=False)`
  - `scene.set_camera_orientation(phi=None, theta=None, gamma=None, zoom=None, focal_distance=None, frame_center=None)`
  - `scene.set_to_default_angled_camera_orientation(...)`
  - `scene.move_camera(start_time=..., duration=..., phi=..., theta=..., gamma=..., zoom=..., focal_distance=..., frame_center=...)`
  - `scene.begin_ambient_camera_rotation(rate=15.0, about="theta", start_time=0.0, duration=None)`
  - `scene.begin_3dillusion_camera_rotation(rate=1.0, origin_phi=None, origin_theta=None, start_time=0.0, duration=None)`

### Core helpers

- If you use `StrokePressure`, import it with `from handanim.core.styles import StrokePressure`.
- `StrokeStyle(color=BLACK, width=1, opacity=1, stroke_pressure=StrokePressure.CONSTANT)`
- `FillStyle(color=BLACK, opacity=1, fill_pattern="hachure", hachure_angle=45, hachure_gap=4, hachure_line_width=1, zigzag_offset=-1, fill_weight=2)`
- `SketchStyle(bowing=1, max_random_offset=2, roughness=1, curve_tightness=0, curve_fitting=0.95, curve_step_count=9, disable_multi_stroke=False, disable_font_mixture=True)`
- `DrawableGroup(elements=[...], grouping_method="parallel")`
  - `parallel`: all children animate together
  - `series`: one event is subdivided across children in order
- Advanced exports: `Drawable`, `AnimationEvent`, `AnimationEventType` are available but usually not instantiated directly.

## Primitive reference

### Lines and curves

- `Line(start=(x1, y1), end=(x2, y2), stroke_style=..., sketch_style=...)`
- `LinearPath(points=[(x1, y1), ...], close=False, stroke_style=..., sketch_style=...)`
- `Arrow(start_point=(x1, y1), end_point=(x2, y2), arrow_head_type="->", arrow_head_size=10.0, arrow_head_angle=45.0, ...)`
  - `arrow_head_type` supports `"->"`, `"->>"`, `"-|>"`
- `CurvedArrow(points=[...], arrow_head_type="->", arrow_head_size=10.0, arrow_head_angle=45.0, ...)`
- `Curve(points=[(x1, y1), (x2, y2), ...], stroke_style=..., sketch_style=...)`

### Shapes

- `Polygon(points=[...], stroke_style=..., fill_style=..., sketch_style=...)`
- `Rectangle(top_left=(x, y), width=w, height=h, ...)`
- `Square(top_left=(x, y), side_length=s, ...)`
- `RoundedRectangle(top_left=(x, y), width=w, height=h, border_radius=0.1, ...)`
  - `border_radius` is a fraction of the smaller side
- `RoundedSquare(top_left=(x, y), side_length=s, border_radius=0.1, ...)`
- `Ellipse(center=(cx, cy), width=w, height=h, ...)`
- `Circle(center=(cx, cy), radius=r, ...)`
- `NGon(center=(cx, cy), radius=r, n=5, ...)`

### 3D primitives

- `ThreeDAxes(x_range=(-4, 4, 1), y_range=(-4, 4, 1), z_range=(-4, 4, 1), x_length=8, y_length=8, z_length=8, include_ticks=True, ...)`
  - Use `axes.c2p(x, y, z)` / `axes.coords_to_point(x, y, z)` for 3D coordinates.
- `Line3D(start=(x1, y1, z1), end=(x2, y2, z2), stroke_style=...)`
- `Prism(dimensions=(dx, dy, dz), center=(x, y, z), stroke_style=..., fill_style=...)`
- `Cube(side_length=2.0, center=(x, y, z), stroke_style=..., fill_style=...)`
- `Sphere(radius=1.0, center=(x, y, z), resolution=(u_steps, v_steps), stroke_style=..., fill_style=...)`
- `Cylinder(radius=1.0, height=2.0, center=(x, y, z), resolution=(u_steps, v_steps), ...)`
- `Cone(base_radius=1.0, height=2.0, center=(x, y, z), resolution=(u_steps, v_steps), ...)`
- `Torus(major_radius=2.0, minor_radius=0.5, center=(x, y, z), resolution=(u_steps, v_steps), ...)`
- `Dot3D(point=(x, y, z), radius=0.08, ...)`
- `ParametricSurface(func=lambda u, v: (x, y, z), u_range=(u0, u1), v_range=(v0, v1), resolution=(u_steps, v_steps), checkerboard_colors=None, ...)`
- `Surface(...)` is an alias for `ParametricSurface(...)`.

### Flowcharts and diagrams

- Exported: `FlowchartNode`, `FlowchartProcess`, `FlowchartDecision`, `FlowchartTerminator`, `FlowchartInputOutput`, `FlowchartConnector`
- Also available: `Table`
- Usually instantiate the specific node classes, not `FlowchartNode` directly.
- Node signature pattern:
  - `FlowchartProcess(text, top_left, width, height, font_size=30, font_name=None, text_padding=18.0, text_stroke_style=None, stroke_style=..., fill_style=..., sketch_style=...)`
- Shape meaning:
  - `FlowchartProcess`: rectangle
  - `FlowchartDecision`: diamond
  - `FlowchartTerminator`: rounded start/end node
  - `FlowchartInputOutput`: slanted parallelogram-style I/O node
- Node helpers:
  - `node.center`
  - `node.anchor_point("top" | "right" | "bottom" | "left" | "center")`
  - `node.connect_to(other, start_side=None, end_side=None, waypoints=None, elbow=None, stroke_style=..., sketch_style=...)`
  - `elbow` supports `"horizontal"` or `"vertical"`
- Direct connector:
  - `FlowchartConnector(start=(x1, y1), end=(x2, y2), waypoints=[...], arrow_head_type="->", arrow_head_size=10.0, arrow_head_angle=45.0, ...)`
- Table helper:
  - `Table(data=[[...], [...]], top_left=(x, y), col_widths=[...], row_heights=[...], font_size=28, cell_padding=12.0, header_rows=1, header_columns=0, header_fill_style=..., stroke_style=..., fill_style=..., sketch_style=...)`
  - Use `fill_style` for normal cells and `header_fill_style` for header rows/columns.
  - Geometry helpers: `table.cell_bbox(row, col)`, `table.cell_center(row, col)`, `table.center`

### Text and math

- `Text(text="Hello", position=(x, y), font_size=48, font_name=None, scale_factor=1.0, rect_box=None, rect_padding=0.0, align="center", line_spacing=1.25, stroke_style=..., sketch_style=..., glow_dot_hint=...)`
- `Math(tex_expression=r"$a^2+b^2=c^2$", position=(x, y), font_size=96, font_name="handanimtype1", stroke_style=..., sketch_style=..., glow_dot_hint=...)`
- Use raw strings for LaTeX math: `r"$...$"`
- Important fonts:
  - `"feasibly"` = best default for handwritten text
  - `"handanimtype1"` = default math font
  - also available: `"headstay"`, `"backstay"`, `"caveat"`, `"permanent_marker"`, `"notosans_math"`
- `text` may contain `\n` for multiline layout.
- `align` supports `"left"`, `"center"`, `"right"`.
- `rect_box=(x, y, w, h)` and `rect_padding` are useful for fitting text inside boxes/nodes; `align` controls horizontal placement inside that box.

### SVG and vector art

- `SVG.from_svg_file("assets/diagram.svg", position=(x, y), stroke_style=..., sketch_style=..., glow_dot_hint=...)`
- `SVG(svg_paths=[...], position=(x, y), ...)`
- `VectorSVG.from_svg_file("assets/diagram.svg", position=(x, y), glow_dot_hint=...)`
- Transform vector objects with `.scale(...)`, `.translate(...)`, `.rotate(...)`.

### Raster media

- `Image(path="assets/photo.png", top_left=(x, y), width=None, height=None, preserve_aspect_ratio=True, opacity=1.0, ...)`
- `Video(path="assets/clip.mp4", top_left=(x, y), width=None, height=None, preserve_aspect_ratio=True, opacity=1.0, scene_start=0.0, source_start=0.0, playback_rate=1.0, loop=True, source_fps=None, ...)`
- Use `FadeInAnimation` / `FadeOutAnimation` for image/video appearance.

### Erasing

- `Eraser(objects_to_erase=[drawable1, drawable2], drawable_cache=scene.drawable_cache, glow_dot_hint={"color": ERASER_HINT_COLOR, "radius": 10})`
- Add it with `SketchAnimation` to get a whiteboard erasing effect.

## Transformations

Transformations return **new drawables**:

- `drawable.translate(offset_x=100, offset_y=50)`
- `drawable.scale(scale_x=1.2, scale_y=1.2)`
- `drawable.rotate(angle=45)`
- `drawable.translate3d(offset_x=1.0, offset_y=0.0, offset_z=0.0)`
- `drawable.scale3d(scale_x=1.2, scale_y=1.2, scale_z=1.2)`
- `drawable.rotate3d(angle=45, axis=(0, 1, 0), center_of_rotation=None)`
- `drawable.move_to3d(point=(x, y, z))`
- Chaining is allowed: `obj.translate(100, 0).rotate(15).scale(1.1, 1.1)`

## Animation reference

All animation constructors support `start_time`, `duration`, optional `easing_fun`, and optional `data`.

- `SketchAnimation(start_time=0, duration=2, data={"wait_before_fill": 0.5})`
  - most common animation
  - progressively draws borders, then fill
  - if a drawable has `glow_dot_hint={"color": ..., "radius": ...}`, the sketch animation shows an animated pen tip automatically
- `FadeInAnimation(start_time=0, duration=1)`
- `FadeOutAnimation(start_time=3, duration=1)`
- `TransformAnimation(target_drawable=target, start_time=2, duration=2, path_arc=0.0)`
- `ReplacementTransformAnimation(target_drawable=target, start_time=4, duration=2, path_arc=0.0)`
- Manim-style compatibility names are also available: `Transform`, `ReplacementTransform`, `FadeTransform`, `FadeTransformPieces`, `ClockwiseTransform`, `CounterclockwiseTransform`
- Stateful transform helpers: `MoveToTarget`, `Restore`, `TransformFromCopy`
- Method/function transforms: `ApplyMethod`, `ApplyFunction`, `ApplyPointwiseFunction`, `ApplyMatrix`, `ApplyComplexFunction`, `ApplyPointwiseFunctionToCenter`
- Spatial helpers: `ScaleInPlace`, `ShrinkToCenter`, `CyclicReplace`, `Swap`
- Style helper: `FadeToColor(color=(r, g, b), start_time=..., duration=...)`
- `ZoomInAnimation(start_time=1, duration=1.5)`
- `ZoomOutAnimation(start_time=5, duration=1.5)`
- `TranslateToAnimation(start_time=2, duration=2, data={"point": (x, y)})`
- `TranslateFromAnimation(start_time=2, duration=2, data={"point": (x, y)})`
- `TranslateToPersistAnimation(start_time=2, duration=2, data={"point": (x, y)})`
- `Translate3DAnimation(offset=(dx, dy, dz), start_time=2, duration=2)`
- `Scale3DAnimation(scale_x=1.2, scale_y=1.2, scale_z=1.2, start_time=2, duration=2, center_of_scaling=None)`
- `Rotate3DAnimation(angle=90, axis=(0, 1, 0), start_time=2, duration=2, center_of_rotation=None)`
- `MoveTo3DAnimation(target=(x, y, z), start_time=2, duration=2)`
- Camera animations for `ThreeDScene`:
  - `MoveCameraAnimation(phi=None, theta=None, gamma=None, zoom=None, focal_distance=None, frame_center=None, start_time=..., duration=...)`
  - `AmbientCameraRotationAnimation(about="theta", rate=15.0, start_time=..., duration=None)`
  - `IllusionCameraRotationAnimation(rate=1.0, origin_phi=None, origin_theta=None, start_time=..., duration=None)`
- `CompositeAnimationEvent(events=[...])`
  - use for sequences like draw -> move -> fade or draw -> zoom + translate
- `TransformAnimation` morphs one drawable into another Manim-style by interpolating aligned shape paths.
- `ReplacementTransformAnimation` performs the same morph but swaps the target drawable into the scene at the end.
- `path_arc` makes morphing points travel on a curved arc instead of a straight line.
- morphing also works on parallel `DrawableGroup` objects, with smart source/target matching by geometry.
- if group sizes differ, unmatched elements can gracefully collapse away or emerge during the morph.
- `drawable.generate_target()` followed by `MoveToTarget(...)` gives Manim-style target-based motion.
- `drawable.save_state()` followed by `Restore(...)` gives reversible stateful motion.

## Best practices for generated code

- Prefer `Scene(width=1920, height=1088, fps=24)` unless the task clearly needs another size.
- Use `ThreeDScene` when the description mentions depth, camera orbit, 3D axes, surfaces, solids, or objects moving in x/y/z.
- Use `FONT_NAME = "feasibly"` for ordinary handwriting unless a different font is requested.
- Use `StrokeStyle(color=BLACK, width=2)` as a safe default for outlines/text.
- Use `SketchStyle(roughness=1 to 3, bowing=1 to 2)` for a natural hand-drawn look.
- Use `FillStyle(..., hachure_gap=8 to 12, opacity=0.2 to 0.5)` for pleasant educational diagrams.
- Use `DrawableGroup` for labels or parts that should animate together.
- In 3D scenes, use `add_fixed_in_frame_mobjects(...)` for screen-space titles/HUDs and `add_fixed_orientation_mobjects(...)` for labels that should follow 3D points while staying upright.
- Use `CompositeAnimationEvent` for multi-step emphasis on the same drawable.
- Prefer `ThreeDAxes`, `Surface`, `Sphere`, `Cube`, `Cylinder`, `Cone`, `Torus`, and `Prism` for 3D scenes instead of faking depth with 2D polygons.
- Keep timings readable: major events usually 0.5 to 3 seconds.
- Use world coordinates with generous spacing; avoid cramped layouts.
- Prefer colors from `handanim.stylings.color` instead of raw RGB unless a custom color is required.
- Common colors available: `BLACK`, `WHITE`, `RED`, `GREEN`, `BLUE`, `CYAN`, `AQUA`, `MAGENTA`, `YELLOW`, `LIGHT_GRAY`, `ERASER_HINT_COLOR`, `GRAY`, `DARK_GRAY`, `ORANGE`, `BROWN`, `SKY_BLUE`, `NAVY`, `TEAL`, `PINK`, `PURPLE`, `LIME`, `PASTEL_RED`, `PASTEL_GREEN`, `PASTEL_BLUE`, `PASTEL_YELLOW`, `PASTEL_ORANGE`, `PASTEL_PURPLE`.
